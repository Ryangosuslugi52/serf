# Этап 3: Реализация клиентской части и фич приложения «Шеф-стол»

**Подход:** Клиентское веб-приложение (Single Page Application), разделенное на логические модули: HTML-структура (`index.html`), стили интерфейса (`style.css`) и бизнес-логика с mock-данными (`script.js`).

## Реализованные фичи согласно брифу

1. **Фича 1 (Динамическое расписание классов):**
   В файле `script.js` зашит массив mock-данных кулинарных классов на 7 дней вперед. На интерфейс выводятся карточки с названиями мастер-классов, именами шефов и их рейтингом. Логика приложения считывает статусы бэкенда и корректно обрабатывает три состояния:
   - Класс доступен (отображается остаток мест, кнопка активна).
   - Класс заполнен / Full State (выводится "Мест нет", кнопка записи заблокирована).
   - Класс отменен студией / Canceled State (карточка визуально приглушается, выводится причина форс-мажора заказчика Артёма, кнопка записи жестко заблокирована).

2. **Фича 2 (Модальное окно записи и прокат инвентаря):**
   При клике на доступный класс открывается интерактивное модальное окно. Реализован чекбокс выбора аренды фартука и ножей. При его переключении JavaScript мгновенно пересчитывает итоговую сумму в DOM (+500 руб. к базовой цене). При подтверждении записи срабатывает локальное списание: количество свободных мест уменьшается на 1, а запись падает в локальный State пользователя.

3. **Фича 3 (Вкладка «Мои бронирования» и оценка шефа):**
   Создан экран личного кабинета клиента. Записанные классы попадают туда в реальном времени. Реализована кнопка «Завершить класс», которая эмулирует окончание мероприятия и открывает интерактивную пятизвездочную шкалу. Пользователь может кликнуть на звезду, отправить оценку шеф-повару и получить фидбек от системы.

## Артефакты ИИ (Промпты для Claude)

Для генерации кодовой базы поочередно использовались три следующих промпта:

### Промпт №1 (HTML-структура):
> "Напиши HTML-структуру (файл index.html) для MVP-приложения кулинарной студии «Шеф-стол». Приложение должно быть оформлено как современный интерфейс под мобильный экран. Разметка должна включать: шапку с переключателем вкладок "Расписание классов" и "Мои бронирования", контейнер для списка карточек мастер-классов (Фича 1), модальное окно бронирования (Фича 2) с формой, чекбоксом для аренды инвентаря ("Аренда фартука и ножей (+500 руб)") и отображением итоговой стоимости, контейнер во вкладке "Мои бронирования" для отображения успешных записей с кнопкой "Завершить класс" и блоком интерактивных звездочек (Фича 3) для оценки шефа. Подключи внешние файлы стилей <link rel="stylesheet" href="style.css"> и скриптов <script src="script.js"></script>. Не пиши CSS и JS внутри этого файла, только чистый HTML5."

### Промпт №2 (CSS-стили):
> "Напиши файл стилей (style.css) для мобильного веб-приложения кулинарной студии. Используй современный, аккуратный дизайн (светлая тема, мягкие тени, скругления углов карточек по 12-16px, приятные шрифты вроде Inter/Roboto, адаптивная верстка в один столбец, имитирующая мобильное приложение). Обязательно стилизуй: вкладки (активная/неактивная), карточки классов для трех состояний (доступен, мест нет, отменен студией), модальное окно (плавное появление по центру экрана, размытие заднего фона/overlay), звездочки рейтинга (интерактивные, меняющие цвет при наведении/активности)."

### Промпт №3 (JS-логика):
> "Напиши JavaScript-код (файл script.js) для управления MVP кулинарной студии. Код должен содержать: массив mock-данных кулинарных классов на 7 дней вперед (минимум 4 класса с разными статусами: один доступен, один заполнен, один отменен студией с текстовой причиной форс-мажора). Каждая сущность должна иметь ID, название, имя шефа, его рейтинг, цену, общее и доступное количество мест. Реализуй: логику рендеринга карточек в зависимости от их статуса; логику модального окна (при клике на "Записаться" сохраняется ID класса, чекбокс аренды фартука пересчитывает цену в DOM на +500 руб); логику подтверждения (places уменьшаются на 1, бронь сохраняется в массив, интерфейс обновляется); логику переключения вкладок; фичу 3 (вкладка "Мои бронирования" показывает записанные классы, кнопка "Завершить класс" открывает оценку шефа на 1-5 звезд, при клике статус сохраняется)."
