# Этап 2: Архитектура приложения и структура Mock-данных

**Концепция:** Клиентское Single Page веб-приложение (SPA). Поскольку реального бэкенда нет, слой данных (Data Layer) эмулируется прямо внутри приложения с помощью локального Mock-провайдера, который отдаёт готовые структуры данных.

## Архитектурные слои клиентской части

1. **Интерфейс (UI Layer):** Чистый HTML5 и CSS3 для отрисовки карточек кулинарных классов, кнопок записи и модального окна бронирования.
2. **Логика и состояние (Business Logic Layer):** Ванильный JavaScript, который управляет состоянием приложения: отслеживает выбранный класс, динамически пересчитывает стоимость аренды инвентаря и обновляет количество доступных мест на экране после записи.
3. **Имитация данных (Mock Data Layer):** JavaScript-объект, содержащий массив кулинарных классов на 7 дней вперёд, имитирующий ответ от "black-box" бэкенда.

---

## Структура данных (JSON-модели)

Для работы приложения используются три ключевые сущности:

### 1. Кулинарный класс (`CookingClass`)
```json
{
  "id": "Int",
  "title": "String (Название мастер-класса)",
  "chef_id": "Int (ID шеф-повара)",
  "datetime": "String (Дата и время проведения)",
  "duration": "String (Длительность)",
  "price": "Int (Базовая цена за участие)",
  "total_places": "Int (Всего мест)",
  "available_places": "Int (Осталось мест)",
  "status": "String ['available' | 'full' | 'canceled_by_studio']",
  "cancel_reason": "String | null (Причина отмены, если статус отменён)",
  "rental_equipment_price": "Int (Стоимость проката фартука и ножей)"
}
```

### 2. Шеф-повар (`Chef`)
```json
{
  "id": "Int",
  "name": "String (Имя шефа)",
  "rating": "Float (Рейтинг от 1.0 до 5.0)"
}
```

### 3. Бронирование (`Booking`)
```json
{
  "id": "Int (ID бронирования)",
  "class_id": "Int (ID класса, на который записался клиент)",
  "with_rental": "Boolean (Выбран ли прокат экипировки: true/false)",
  "total_price": "Int (Итоговая стоимость с учетом проката)",
  "chef_rating": "Int | null (Оценка шефу, выставляется после эмуляции посещения класса)"
}
```

## Артефакты ИИ (Промпты)
Для генерации данных требований к ИИ был отправлен следующий промпт:
> "Действуй как Software Architect. Спроектируй структуру mock-данных в формате JSON для веб-приложения кулинарной студии. Нам нужно описать три сущности: модель кулинарного класса (с ценой, местами, прокатом фартуков и статусами отмены студией), модель шеф-повара с рейтингом и модель самого бронирования (с флагом аренды и оценкой шефу). Покажи схему данных и опиши простую архитектуру для фронтенда без реального бэкенда."
