# Этап 1: Анализ и требования к MVP кулинарной студии «Шеф-стол»

**Цель документа:** Описать минимально жизнеспособный продукт (MVP) для клиентского веб-приложения кулинарной студии на основе брифа заказчика[cite: 2]. 

## Пользовательские истории (User Stories)

1. **Просмотр расписания:** 
   Я, как клиент кулинарной студии, хочу видеть расписание мастер-классов на ближайшие 7 дней, чтобы выбрать удобное время для посещения[cite: 2].
2. **Детали класса:** 
   Я, как клиент, хочу видеть тему занятия (меню) и имя шеф-повара с его рейтингом, чтобы записаться к лучшему мастеру на интересующее меня блюдо[cite: 2].
3. **Прокат инвентаря:** 
   Я, как клиент, хочу иметь возможность выбрать при бронировании опцию «нужен прокат фартука и ножей», чтобы не везти свои вещи в студию[cite: 2].
4. **Статус отмены:** 
   Я, как клиент, хочу сразу видеть в расписании, если класс «Отменён студией» из-за форс-мажора, чтобы знать причину и не пытаться на него записаться[cite: 2].
5. **Оценка шефа:** 
   Я, как клиент, хочу иметь возможность поставить оценку шеф-повару по пятизвездочной шкале после посещения занятия, чтобы помочь студии оценивать работу сотрудников[cite: 2].

## Основные пользовательские сценарии (User Flows)

### Сценарий 1: Успешная запись на класс с прокатом экипировки
1. Клиент открывает главное окно приложения.
2. Система отображает список доступных кулинарных классов на ближайшие 7 дней[cite: 2].
3. Клиент выбирает интересующий класс (например, «Итальянская паста») и нажимает кнопку «Записаться»[cite: 2].
4. Открывается окно оформления записи, где клиент видит базовую стоимость[cite: 2].
5. Клиент отмечает галочкой пункт «Нужен прокат фартука и ножей»[cite: 2].
6. Система динамически пересчитывает итоговую стоимость, добавляя цену проката[cite: 2].
7. Клиент нажимает кнопку «Подтвердить запись».
8. Система выводит сообщение об успешном бронировании, а количество доступных мест на этот класс уменьшается на 1[cite: 2].

### Сценарий 2: Отображение класса, отмененного студией из-за форс-мажора
1. Клиент открывает список классов на неделю[cite: 2].
2. В списке отображается класс со статусом «Отменён студией»[cite: 2].
3. Рядом со статусом выводится текстовая причина отмены (например, «Сорвалась поставка продуктов»)[cite: 2].
4. Кнопка «Записаться» для этого класса визуально заблокирована и не нажимается[cite: 2].
5. Повторная или принудительная запись на этот слот невозможна[cite: 2].

Для генерации данных требований к ИИ был отправлен следующий промпт:
> "Действуй как системный аналитик. Сгенерируй требования к MVP мобильного/веб-приложения для кулинарной студии на основе брифа владельца Артёма[cite: 2]. Учти ограничения: делаем только клиентскую часть, бэк — black-box, выводим слоты на 7 дней вперед, обрабатываем отмену класса студией, добавляем прокат фартуков и оценку шефа[cite: 2]. Результат выведи в формате User Stories и User Flows."
