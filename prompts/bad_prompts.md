# «Плохие» промпты (антипримеры)

> Учебные материалы для демонстрации типичных ошибок при работе с ИИ (промпт-инжиниринг). Показывают, как отсутствие контекста, ролей и жестких рамок приводит к генерации избыточного, неподходящего кода или нерелевантных требований.

---

## Антипример 1: Запрос ТЗ «в лоб» без контекста MVP

**Промпт:**
> Напиши техническое задание для сайта кулинарной студии.

**Что не так:**
* **Отсутствие рамок MVP:** ИИ не знает, что мы проектируем минимальный прототип клиентской части Single Page Application.
* **Шаблонный мусор:** Нейросеть начинает выдумывать тяжелую инфраструктуру, интеграции со сторонними сервисами и модули автоматизации бизнеса, которые физически невозможно и не нужно реализовывать в рамках текущего учебного задания.

**Ответ ИИ (фрагмент нерелевантного ответа):**
> *«...Система автоматизации кулинарной студии должна включать: Личный кабинет Администратора для заведения расписания, модуль складского учёта остатков продуктов на базе 1С:Предприятие для шеф-поваров, интеграцию с кассовыми аппаратами Эвотор по 54-ФЗ для печати чеков при оплате наличными, а также модуль отправки SMS-уведомлений через шлюз SMS.ru...»*

**Как доводить:**
Необходимо сразу задавать жесткие рамки: указать, что бэкенд — это "black-box" (скрыт от нас), авторизация и админка отсутствуют, а само приложение представляет собой легкий клиентский интерфейс для просмотра расписания мастер-классов строго на 7 дней вперед.

## Антипример 2: Размытые требования к коду и путаница в технологиях

**Промпт:**
> Напиши код для бронирования мест на кулинарный класс, чтобы всё сохранялось в базу данных.

**Что не так:**
* **Технологический хаос:** В запросе не указан стек технологий. ИИ по умолчанию начинает проектировать полноценный бэкенд-сервер с подключением к реальной базе данных (MongoDB/PostgreSQL), полностью игнорируя факт того, что наше MVP должно работать на чистом JavaScript (Vanilla JS) в одном браузере на Mock-данных.

**Ответ ИИ (фрагмент неподходящего кода):**
> ```javascript
> import express from 'express';
> const mongoose = require('mongoose');
> const app = express();
> 
> app.post('/api/booking', async (req, res) => {
>   const newBooking = new Booking(req.body);
>   await newBooking.save();
>   res.status(201).json({ success: true, message: 'Бронь сохранена в базу данных' });
> });
> ```

**Как доводить:**
Четко специфицировать стек (HTML5, CSS3, Vanilla JS) и явно указывать ИИ, что база данных эмулируется внутри фронтенд-кода в виде локального JS-массива состояний, который должен динамически обновляться без перезагрузки страницы.
