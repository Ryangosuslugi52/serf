# Бизнес-требования

> Этап 2. Зачем бизнесу продукт: цели, метрики успеха, ограничения верхнего уровня.
> Данный документ подготовлен в рамках учебного технического задания для проверяющих.

## 1. Общие положения
* **Цель проекта:** Создание клиентского Single Page веб-приложения (SPA) в рамках MVP для автоматизации онлайн-записи на кулинарные мастер-классы.
* **Стек технологий:** Разработка ведется с использованием HTML, CSS, JavaScript и JSON для хранения Mock-данных.
* **Границы проекта:** В текущую поставку MVP входит исключительно роль Клиента. Интерфейсы администратора, добавление новых шеф-поваров и редактирование расписания находятся вне скоупа текущей поставки.

## 2. Бизнес-требования (Business Requirements)
| ID | Требование | Приоритет | Источник |
| :-- | :-- | :-- | :-- |
| BR-1 | Исключить путаницу и овербукинг мест на кухне студии при бронировании мастер-классов за счет автоматического учета свободных мест. | Must | Учебное задание |
| BR-2 | Предоставить клиентам интерфейс для самостоятельного выбора и онлайн-записи на доступные мастер-классы. | Must | Учебное задание |
| BR-5 | Фиксировать статусы отмены записей для оптимизации закупки продуктов и загрузки станций. | Must | Учебное задание |
| BR-7 | Реализовать минимально жизнеспособный продукт (MVP) в установленные жесткие сроки на базе Mock-данных. | Must | Учебное задание |
| BR-8 | Расчет стоимости производится в приложении, однако оплата на этапе MVP остается офлайн в студии. | Must | Ограничения MVP |

## 3. Метрики успеха (Success Metrics)
| ID | Метрика | Целевой ориентир | Связь с BR |
| :-- | :-- | :-- | :-- |
| M-1 | Доля онлайн-записей через веб-интерфейс от общего числа клиентов. | ≥ 70% | BR-2 |
| M-2 | Количество инцидентов с двойным бронированием одного и того же места. | 0 (полное устранение) | BR-1 |
| M-4 | Процент поздних отмен бронирований. | ≤ 10% | BR-5 |
| M-6 | Успешный запуск MVP веб-приложения в рамках установленного учебного дедлайна. | 100% готовность MVP | BR-7 |

## 4. Ограничения верхнего уровня
* Архитектура приложения строится как SPA на чистом JavaScript (Vanilla JS) без использования бэкенд-фреймворков.
* Все данные (классы, шеф-повара, бронирования) эмулируются локально через JSON / Mock Data Layer.
* Модули аналитики, отзывов и оплаты вынесены во вторую фазу разработки и эмулируются на уровне интерфейса.
