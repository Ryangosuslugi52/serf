# Функциональные требования

> Этап 2. Что система должна делать для выполнения бизнес-целей.
> Распределение приоритетов выполнено по методологии MoSCoW (Must, Should, Could, Won't).

## 1. Границы функциональности (Клиентское SPA)
Приложение представляет собой Single Page Application, где вся логика обработки данных выполняется на стороне клиента через JavaScript-классы и Mock-провайдер.

## 2. Авторизация пользователя
* **FR-1:** Система должна обеспечивать легкую регистрацию нового клиента с вводом имени и номера телефона.
* **FR-2:** Система должна авторизовать существующего клиента по номеру телефона.
* **FR-43:** В рамках MVP авторизация эмулируется локально (ввод любого проверочного кода без реальной отправки SMS).

## 3. Просмотр и фильтрация мастер-классов
* **FR-9:** Система должна отображать список предстоящих кулинарных классов (`CookingClass`) на ближайшие 7 дней с указанием названия, даты, времени, длительности, цены, свободных мест и стоимости аренды инвентаря.
* **FR-9a:** В карточке класса должны отображаться данные шеф-повара (`Chef`): его имя и текущий рейтинг.
* **FR-38:** Система должна позволять фильтровать список классов по идентификатору шеф-повара (`chef_id`), доступности свободных мест и датам.
* **FR-44:** Если класс отменен администрацией (`status == 'canceled_by_studio'`), система должна выводить причину отмены (`cancel_reason`) и блокировать кнопку записи.

## 4. Запись и расчет стоимости
* **FR-10:** Система должна позволять авторизованному клиенту оформить бронирование на выбранный класс.
* **FR-11:** При бронировании клиент может выбрать опцию аренды экипировки (фартук и ножи) с помощью флага `with_rental`.
* **FR-13:** После успешной записи система должна автоматически уменьшать количество доступных мест (`available_places`) в объекте класса на 1.
* **FR-15:** Система должна аппаратно блокировать запись, если мест не осталось (`available_places == 0`).

### Модель расчета цены (Price Model)
Расчет итоговой стоимости (`total_price`) выполняется слоем бизнес-логики JavaScript по формуле:
* Если `with_rental == true`: `total_price = price + rental_equipment_price`.
* Если `with_rental == false`: `total_price = price`.
Валюта расчетов — российские рубли (RUB).

## 5. Личный кабинет и история
* **FR-35a:** Система должна выводить в личном кабинете список всех бронирований текущего пользователя (`Booking`) с указанием статусов.
* **FR-16:** Система должна позволять клиенту отменить активное бронирование, при этом `available_places` класса увеличивается на 1.
* **FR-50:** В списке прошедших мастер-классов система должна предоставлять клиенту возможность выставить оценку шеф-повару (`chef_rating`) от 1 до 5.
